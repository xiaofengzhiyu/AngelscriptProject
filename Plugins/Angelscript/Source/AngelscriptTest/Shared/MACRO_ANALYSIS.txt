CURRENT OBSERVATIONS
--------------------

1. The active macro family is `ASTEST_*`, not `ANGELSCRIPT_TEST` or `ANGELSCRIPT_ISOLATED_TEST`.
2. Current migration work keeps `IMPLEMENT_SIMPLE_AUTOMATION_TEST(...)` for registration and uses `ASTEST_CREATE_ENGINE_*` plus `ASTEST_BEGIN_*` / `ASTEST_END_*` inside `RunTest(...)` when that pairing is semantically safe.
3. Existing helper names such as `GetOrCreateSharedCloneEngine()`, `AcquireCleanSharedCloneEngine()`, and `AcquireFreshSharedCloneEngine()` should be mapped carefully because their runtime semantics differ.
4. Validation and migration decisions should be based on actual helper behavior in `AngelscriptTestUtilities.h`, not on historical helper naming.
